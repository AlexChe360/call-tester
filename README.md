# call-tester — тестирование звонков модем-на-модем (Go)

Автоматизация голосовых вызовов между USB-модемами на Raspberry Pi 5.
Формирует CDR-отчёты для сверки с биллингом.

## Быстрый старт

### Вариант 1: Кросс-компиляция на маке (рекомендуется)

```bash
# На маке — установить Go если ещё нет
brew install go

# Клонировать/скопировать проект
cd call-tester-go

# Скачать зависимости
go mod tidy

# Собрать под RPi 5 и задеплоить
make deploy
```

Одна команда `make deploy` соберёт бинарник под ARM64, скопирует его и конфиги на RPi по SSH.

### Вариант 2: Сборка прямо на RPi

```bash
# Установить Go на RPi
sudo apt install -y golang

# Собрать
cd call-tester-go
go mod tidy
go build -o call-tester ./cmd/call-tester
```

### Настройка

```bash
# На RPi — определить порты модемов
chmod +x setup_rpi.sh
sudo ./setup_rpi.sh

# Отредактировать конфиг — вписать порты и номера SIM
nano config.yaml
```

### Использование

```bash
# Проверить все модемы
./call-tester check

# Одиночный тестовый звонок (sim7600 -> ec25_1, 15 секунд)
./call-tester call sim7600 ec25_1 -d 15

# Запустить полный сценарий
./call-tester run scenarios/full_matrix.yaml -o reports/
```

## Формат CSV-отчёта

```
ID,Сценарий,Шаг,Модем_А,Номер_А,Модем_Б,Номер_Б,Направление,Дата_начала,Время_начала,Время_ответа,Время_конца,Длительность_разговора_сек,Полная_длительность_сек,Статус
```

Ключевые поля для сверки с биллингом:
- **Номер_А** / **Номер_Б** — участники
- **Время_ответа** — момент соединения (биллинг обычно тарифицирует с этого момента)
- **Длительность_разговора_сек** — для сравнения с тарифицированной длительностью

## Команды

| Команда | Описание |
|---------|----------|
| `check` | Проверить подключение всех модемов, сигнал, регистрацию |
| `run <file.yaml>` | Запустить сценарий из YAML-файла |
| `call <from> <to>` | Одиночный звонок между двумя модемами |
| `example-config` | Показать пример config.yaml |

## Определение AT-портов

```bash
# Подключите модемы по одному и смотрите новые порты
ls /dev/ttyUSB*

# Проверяйте — AT-порт ответит OK
echo -e "AT\r" > /dev/ttyUSB2 && timeout 1 cat /dev/ttyUSB2
```

Типичные AT-порты:
- **SIM7600**: создаёт 5 портов, AT = `ttyUSBx+2`
- **EC25/EP06**: создаёт 4 порта, AT = `ttyUSBx+2` или `ttyUSBx+3`

## Справочник AT-команд
 
### Базовые
 
| Команда | Описание |
|---------|----------|
| `AT` | Проверка связи с модемом (ответ: OK) |
| `ATE0` | Выключить эхо команд |
| `ATE1` | Включить эхо команд |
| `ATI` | Информация о модеме (модель, ревизия) |
| `AT+CMEE=2` | Включить расширенные текстовые ошибки |
| `AT+CFUN?` | Режим функциональности (1 = полный) |
| `AT+CFUN=1` | Перезагрузить модем в полный режим |
 
### SIM-карта
 
| Команда | Описание |
|---------|----------|
| `AT+CPIN?` | Статус PIN-кода (READY = разблокирована) |
| `AT+CPIN="1234"` | Ввести PIN-код |
| `AT+CNUM` | Показать номер SIM (не все модемы поддерживают) |
| `AT+CCID` | Показать ICCID SIM-карты |
| `AT+CIMI` | Показать IMSI |
 
### Сеть и сигнал
 
| Команда | Описание |
|---------|----------|
| `AT+CREG?` | Статус регистрации (0,1 = зарегистрирован, 0,5 = роуминг) |
| `AT+CSQ` | Уровень сигнала (RSSI,BER). RSSI: 0-31 (чем больше тем лучше), 99 = нет |
| `AT+COPS?` | Текущий оператор |
| `AT+COPS=?` | Список доступных операторов (долгая команда, ~30с) |
| `AT+QNWINFO` | Тип сети и частота (Quectel) |
| `AT+CPSI?` | Системная информация (SIM7600) |
 
### Голосовые вызовы
 
| Команда | Описание |
|---------|----------|
| `ATD+7XXXXXXXXXX;` | Набрать номер (`;` = голосовой вызов, обязательна!) |
| `ATA` | Ответить на входящий звонок |
| `ATH` | Положить трубку |
| `AT+CHUP` | Положить трубку (альтернатива, надёжнее на некоторых модемах) |
| `AT+CLCC` | Список активных вызовов (статус, номер, направление) |
| `AT+CLIP=1` | Включить определитель номера для входящих |
| `ATS0=1` | Автоответ после 1 гудка |
| `ATS0=0` | Выключить автоответ |
| `AT+VTD=5` | Длительность DTMF-тона (1-255, единица = 100мс) |
| `AT+VTS="1"` | Отправить DTMF-тон (цифры 0-9, *, #) |
 
### Расшифровка +CLCC
 
`+CLCC: idx,dir,stat,mode,mpty,"number",type`
 
| Поле | Значение |
|------|----------|
| `dir` | 0 = исходящий, 1 = входящий |
| `stat` | 0 = active, 1 = held, 2 = dialing, 3 = alerting, 4 = incoming, 5 = waiting |
| `mode` | 0 = voice, 1 = data, 2 = fax |
 
### SMS (опционально, для уведомлений)
 
| Команда | Описание |
|---------|----------|
| `AT+CMGF=1` | Текстовый режим SMS |
| `AT+CMGS="+7XXXXXXXXXX"` | Отправить SMS (после ввода текста — Ctrl+Z) |
| `AT+CMGL="ALL"` | Прочитать все SMS |
 
### USSD
 
| Команда | Описание |
|---------|----------|
| `AT+CUSD=1,"*100#",15` | Отправить USSD-запрос (напр. баланс) |
 
### Расшифровка уровня сигнала (AT+CSQ)
 
| RSSI | dBm | Качество |
|------|-----|----------|
| 0-9 | -113...-95 | Плохой |
| 10-14 | -93...-85 | Слабый |
| 15-19 | -83...-75 | Нормальный |
| 20-24 | -73...-65 | Хороший |
| 25-31 | -63...-51 | Отличный |
| 99 | — | Нет сигнала |

## Частые проблемы

**Permission denied** → `sudo usermod -aG dialout $USER` + перелогиниться

**Модем не отвечает** → Неправильный порт. Пробуйте соседние ttyUSBx.

**Не зарегистрирован в сети** → SIM не активна или нужен PIN: `AT+CPIN?`

**Звонок не проходит** → Проверьте голосовой сервис на SIM и сигнал: `AT+CSQ`
